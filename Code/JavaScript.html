JavaScript Historik

JavaScript är ett webbspråk som fungerar som ett scriptspråk. Java genererar inte en miljö, utan utvecklar finesserna i en redan existerande och då programmerar Java arbetssättet (”beteendet”) hos till exempel en webbplats. JavaScript kan tillexempel kontrollera ifall alla formulär är ifyllda i en webbsida innan de skickas in. JavaScript kan även byta annonser på en webbplats eller dölja vissa delar av webbplatsen. Kortfattat kan man säga att JavaScript ansvarar för möblering av webbplatsen (1–4).

JavaScript började utvecklas under namnet Mocha men sedan bytte namn till Livescript. Tre månader efter utgivning bytte den namn till JavaScript av marknadsföringsskäl. Detta på grund av att det ungefär samtidigt som JavaScript börjat utvecklas så började Netscape gå i samarbete med Sun Microsystems och använda deras program Java. Efter bestämde sig Netscape för att använda JavaScript som scriptspråk.

Grundaren för JavaScript är Brendan Eich och den första utgåvan gavs ut 1995 under namnet Livescript. Det dröjde inte länge innan även Microsoft skapade sin egen variant under namnet ”Jscript”. Detta ledde till att det blev svårt att få en webbplats att se likadan ut i alla webbläsare (5). Netscape skickade då ut JavaScript till ECMA för att använda den som en standard för alla webbläsare. ECMA är en organisation som standardiserar och utvecklar tekniska idéer som kommer från olika företag. Det ledde till den första utgåvan för ECMAscript som kom ut 1997. Andra utgåvan kom ut 1998 och tredje ECMAscript kom ut 1999. Medan ECMAscript utgavs så jobbade Microsoft upp sig och kort därefter blev Internet Explorer den mest använda webbläsaren som använde Jscript som standard. Till slut anslöt sig Microsoft till ECMA men det blev inte långvarigt och det ledde till att ECMAscript 4 upplöstes i början på 2000-talet (5,6).

Det dröjde till 2004 innan Mozilla kom ut med firefox som blev väldigt populär bland internetklienterna. Det ledde till att Mozilla 2005 anslöt sig till ECMA och de började vidareutveckla ECMAScript for XML och som började användas som standard för Adobe System. Men även här kunde de olika parterna inte komma överens och ECMAscript 4 misslyckades. Samtidigt som det här sker så skrev Jesse James Garrett sin bok som förklarade vikten av JavaScript. Det ledde till ett större intresse hos utvecklarna efter att han även nämnde att tack vare JavaScript så behövde man inte ladda om hemsidan. Även Google släppte år 2008 sin webbläsare Crome som använde JavaScript som sin sökmotor. Detta ledde till att en konferens hölls samma år och man bestämde att ECMAscript 5 skulle släppas som standard (5,6). 2015 så publicerades ECMAscript 6 med en rad nya förändringar och från 2016 så publiceras en ny uppdatering varje år och nu har vi ECMAscript 11 med en rad nya förändringar så som att man kan exportera modulnamnsutrymme samt dynamisk import (7–10). Ibland kallar man den bara för ES då det uppgraderas ofta och då byter versionnamn (4).

Fördelarna med JavaScript är att du kan skapa effekter, visa datum eller skapa möjlighet för att skicka mejl via Webbplatsen (4–5).

Nackdelarna kan vara att JavaScript inte ger fullt stöd för WAI-ARIA (Specifikation för tolkning av element) och att säkerheten kan brista ifall det är samma webbursprung och samma sandlåda kan man utnyttja det och köra en kod i någon annas dator via webben. Därför kan det vara dåligt att lagra lösenord i JavaScript då det är ganska enklat för en angripare att komma åt det (5,11).
